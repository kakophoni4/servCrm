# Матрица ролей

На MVP: **4 фиксированные роли**, без конструктора прав.
Права всех функций настраивает только владелец (фаза 5+; на MVP — хардкод матрицы).

## Роли

| Роль | Код | Описание |
|------|-----|----------|
| Диспетчер | `DISPATCHER` | Создание заявок, претензии, повтор/гарантия |
| Администратор | `ADMIN` | Все заявки/клиенты, мастер, статусы, оплаты |
| Директор | `DIRECTOR` | Как админ + отчёты/касса (ограничен приход) |
| Владелец | `OWNER` | Полный доступ ко всем операциям |

> На MVP директор и владелец работают как админ (+ закладки «скоро»). Отчёты и касса — с фазы 3.

## Матрица

| Функция | Диспетчер | Админ | Директор | Владелец |
|---------|-----------|-------|----------|----------|
| Создать/редактировать заявку | ✅ | ✅ | ✅ | ✅ |
| Все заявки / все клиенты | ✅ все* | ✅ | ✅ | ✅ |
| Назначить мастера | ❌ | ✅ | ✅ | ✅ |
| Статусы заявки | ограничено** | ✅ | ✅ | ✅ |
| Претензии CRUD | ✅ | ✅ | ✅ | ✅ |
| Касса приход | — | ✅ | ❌ | ✅ |
| Касса расход (ограниченный) | — | частично*** | ✅ | ✅ |
| Отчёты | ❌ | ❌ | ✅ | ✅ |
| Сотрудники / права / имущество | ❌ | просмотр | частично | ✅ |
| Настройки % ЗП, города | ❌ | ❌ | ✅ категории | ✅ всё |

\* **Решение (день 0):** диспетчер видит **все заявки** (не только свои). См. `10-OPEN-QUESTIONS.md`.

\*\* Диспетчер: создание + комментарий + повтор/гарантия + флаг претензии. Смена статусов исполнения (В пути / В работе / Готов и т.д.) — только админ+.

\*\*\* Админ расходы: только «зп промоутера» и «операционные расходы»; остальное — директор/владелец.

## Auth (MVP)

- Логин: формат `login` (без почты)
- Пароль: bcrypt
- Обязательные поля при регистрации: login, password, ФИО, роль
- Опционально на MVP: телефон, город
- Паспорт/договор/шифропаспорта — вне MVP
