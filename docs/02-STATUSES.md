# Статусы заявки (машина состояний)

```
Не оформлена → Ожидает → В пути → В работе
                                    ├→ В работе СД → В работе → …
                                    ├→ Готов
                                    ├→ Отказ
                                    ├→ Отмена КЦ
```

## Таблица статусов

| Статус | Код | Кто ставит | Условие |
|--------|-----|------------|---------|
| Не оформлена | `NOT_SCHEDULED` | система / диспетчер | нет даты выполнения |
| Ожидает | `WAITING` | система | создана; мастер не выбран / выбран, но не выехал |
| В пути | `ON_THE_WAY` | админ / мастер (бот) | мастер выехал |
| В работе | `IN_PROGRESS` | админ / мастер | начало работ |
| В работе СД | `IN_PROGRESS_SD` | админ / мастер | продолжение; нужна расписка (бот, фаза 4) |
| Готов | `DONE` | админ / мастер | заказ завершён; при сумме >500 — чеки |
| Отказ | `REFUSAL` | админ | отказ от оплаты |
| Отмена КЦ | `CANCELLED_CC` | админ / диспетчер | отмена клиентом |

## Флаги (отдельно от статуса)

| Флаг | Поле | Описание |
|------|------|----------|
| Претензия | `is_claim` | заявка отмечена как претензионная |
| Гарантия | `is_warranty` / тип `WARRANTY` | гарантийный заказ |
| Повтор | `is_repeat` / тип `REPEAT` | повторный заказ (новая заявка) |
| Профильная | `is_profile` | профильная / непрофильная |

## Переходы (MVP web)

| Из | В | Кто |
|----|---|-----|
| `NOT_SCHEDULED` | `WAITING` | система (когда задана дата) |
| `WAITING` | `ON_THE_WAY` | админ |
| `ON_THE_WAY` | `IN_PROGRESS` | админ |
| `IN_PROGRESS` | `IN_PROGRESS_SD` | админ |
| `IN_PROGRESS_SD` | `IN_PROGRESS` | админ |
| `IN_PROGRESS` / `IN_PROGRESS_SD` | `DONE` | админ (+ правило чеков >500 с фазы 2) |
| любой рабочий | `REFUSAL` | админ |
| любой до Готов | `CANCELLED_CC` | админ / диспетчер |

## Правило «Готов» + чеки

Если `paid > 500` → обязательны подтверждающие документы; сумма сохраняется, статус `DONE` — нет.
**Решение дня 0:** правило активно с фазы 2; в MVP поля оплаты есть, блокировка мягкая (предупреждение).
