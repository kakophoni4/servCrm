# Brief — CRM (замена Битрикса)

## Цель

Дать диспетчеру и администратору рабочий контур заявок/клиентов
вместо Битрикса за 2–3 недели.

## Пользователи MVP

- **Диспетчер**: создаёт/видит заявки, претензии, перевод в повтор/гарантию
- **Администратор**: все заявки/клиенты, назначает мастера, статусы, оплаты
- **Директор / Владелец**: как админ + позже отчёты/касса (вне MVP)

## В скоупе MVP

- Auth (login/password, ФИО)
- Заявки + авто ID
- Карточка клиента + история
- Статусы + мастер
- Источник: наша (Авито/листовка) / партнёрская
- Повтор / гарантия / флаг претензии
- Базовые поля оплаты (без жёсткой блокировки «Готов» по чекам — опционально, см. OPEN-QUESTIONS)

## Вне скоупа MVP

- Касса, инкассация, полный набор отчётов
- Бот TG/Max, чат-центр
- Шифрование паспортов, имущество филиалов
- Конструктор прав

## Успех

Диспетчер создаёт заявку за <2 мин; админ назначает мастера и ведёт статус;
клиентская история видна в карточке и в заявке.

## Сроки и бюджет (ориентир)

| Этап | Срок | Бюджет |
|------|------|--------|
| MVP | 2–3 нед | 150–350 тыс ₽ |
| Деньги + претензии | +2–3 нед | +200–350 тыс |
| Касса + отчёты | +3–4 нед | +300–500 тыс |
| Бот TG | +2–3 нед | +150–300 тыс |

## Стек

- Frontend: Next.js + TypeScript
- Backend: NestJS + Prisma
- DB: PostgreSQL 16
- Files: локально `uploads/` → потом S3
- Auth: JWT/session + bcrypt
- Deploy: Docker Compose на VPS
