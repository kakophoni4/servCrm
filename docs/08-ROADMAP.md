# Roadmap и Definition of Done

## Фаза 1 — MVP (2–3 нед) — DoD

- Auth + 4 роли
- CRUD заявок/клиентов, ID, статусы, мастер
- Источники, повтор/гарантия, претензии
- Деплой HTTPS + бэкап БД
- Приёмка по `07-ACCEPTANCE.md`

## Фаза 2 — Деньги + документы (+2–3 нед)

- Документы к заявке
- Жёсткое правило «Готов» при paid > 500
- Категории ЗП + автопересчёт work_sum / to_company

## Фаза 3 — Касса + отчёты (+3–4 нед)

- Касса приход/расход/инкассация
- Расчёт мастеров (двойное подтверждение)
- Отчёты: закрытые / отмены / касса / (черновик рекламы)

## Фаза 4 — Бот TG (+2–3 нед)

- Уведомления, статусы мастера, файлы, «обо мне»
- Дневной отчёт рекламы

## Фаза 5 — Расширения

- Чат-центр TG/Max, имущество, уволенные/восстановление, шифропаспорта

## План работ Фаза 1 (по дням)

| Дни | Задачи | Результат |
|-----|--------|-----------|
| 0 | BRIEF, роли, статусы, ID, справочники, репо, VPS | старт без вопросов по ядру |
| 1 | Каркас Next/Nest, auth, роли, layout | логин работает |
| 2–3 | Client + Order CRUD, ID, список заявок | диспетчер создаёт заявку |
| 4 | Карточка заявки, статусы, назначение мастера | админ-контур |
| 5 | Карточка клиента + история | склейка клиентов |
| 6 | Источники, типы, повтор/гарантия, флаг претензии | логика как в ТЗ |
| 7 | Претензии + поля оплаты | данные для денег |
| 8 | Мастера/сотрудники MVP, сиды городов | справочники |
| 9 | Деплой VPS, HTTPS, бэкап БД | пилот на сервере |
| 10 | Правки по пилоту, приёмка | MVP сдан |

Запас 4–5 дней на баги и уточнения ID/ролей.

## Не трогать до приёмки MVP

Кассу, бота, чат-центр, имущество, шифропаспорта.
