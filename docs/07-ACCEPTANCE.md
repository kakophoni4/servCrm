# Критерии приёмки MVP

Чеклист пилота (день 9–10):

- [ ] Диспетчер логинится и создаёт заявку со всеми полями MVP
- [ ] ID генерируется по утверждённому правилу (`06-ID-RULES.md`), без дублей
- [ ] Клиент находится/создаётся по телефону, история видна
- [ ] Админ назначает мастера и меняет статусы по таблице (`02-STATUSES.md`)
- [ ] Повтор создаёт новую заявку; гарантия помечает доп.статус
- [ ] Претензию можно создать/отметить
- [ ] Список заявок и клиентов на отдельных вкладках
- [ ] Деплой по HTTPS, данные переживают рестарт
- [ ] Роли не дают диспетчеру чужие админ-действия (назначение мастера, полный набор статусов исполнения)

## Поля заявки MVP (обязательный набор)

- имя клиента + телефон (суммарно уникальные)
- тип: новый / гарантия / повторный
- дата/время формирования (авто)
- дата/время назначенного выполнения (nullable → Не оформлена)
- адрес
- возрастная категория
- комментарий
- источник: Наша (Авито/листовка) / Партнёрская (по названию)
- флаг претензии
- тип техники (текст)
- профильная да/нет
- поля оплаты: оплачено / предоплата / комплектующие / есть-нет

## Пилотные пользователи

1 диспетчер + 1 админ + владелец.
